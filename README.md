## 📖 О проекте

**Restik** - современный фронтенд для ресторана европейской кухни. Приложение позволяет гостям:

✨ **Бронировать столики** онлайн с выбором даты, времени и места  
✨ **Просматривать меню** с фото, описаниями и актуальными ценами  
✨ **Делать предзаказ** блюд при оформлении брони  
✨ **Управлять бронированиями** через личный профиль  
✨ **Авторизоваться** с сохранением сессии через JWT

Проект разработан на **чистом JavaScript (Vanilla JS)** без тяжелых фреймворков — это обеспечивает мгновенную загрузку, отличную производительность и простоту поддержки.

## ✨ Возможности

### 🎯 Для гостей:
- 🔐 Регистрация и вход с запоминанием сессии
- 📅 Календарь с валидацией дат (нельзя выбрать прошлое)
- ⏰ Выбор времени с интервалом 30 минут
- 🪑 Визуальная карта зала с отображением свободных/занятых столиков
- 🍴 Предзаказ блюд из меню с подсчётом суммы
- 👤 Личный кабинет с историей бронирований
- 📱 Полностью адаптивный дизайн (мобильные, планшеты, десктоп)

### 🔐 Технические особенности:
- **Аутентификация**: JWT + куки + localStorage (гибкое хранение)
- **Защита маршрутов**: автоматическая проверка токена перед запросами
- **Обработка ошибок**: дружелюбные сообщения вместо технических кодов
- **Конфигурация API**: поддержка переменных окружения через `window.API_CONFIG` 
- **Умные редиректы**: корректные относительные пути на всех страницах

## ⚙️ Конфигурация

### 🌐 Настройка API

Проект использует Directus как бэкенд. По умолчанию указан тестовый туннель:

```javascript
// В браузере, до загрузки страницы:
window.API_CONFIG = {
    DIRECTUS_URL: 'https://your-directus-instance.directus.app'
};
```

Или отредактируйте `config/api-config.js`:

```javascript
const API_CONFIG = {
    DIRECTUS_URL: 'https://your-url.here'
};
```

### Требуемые коллекции в Directus

| Коллекция | Поля | Описание |
|-----------|------|----------|
| menu_items | id, name, description, price, category, image, status | Блюда меню |
| tables | id, name, seats, status (active/occupied/maintenance) | Столики зала |
| reservations | id, date, time, table, user_created, party_size, total, status | Бронирования |
| users | (стандартные Directus) | Пользователи |

### 🔐 Аутентификация

- Токен хранится в куки (30 дней) + localStorage (резерв)
- При сетевых ошибках пользователь не разлогинивается автоматически
- Проверка токена происходит через `/users/me` эндпоинт Directus

## 🔌 API

### 🔹 Эндпоинты Directus

```javascript
// 🔐 Аутентификация
POST /auth/login           # Вход: { email, password } → { access_token, data }
POST /auth/refresh         # Обновление токена
POST /auth/logout          # Выход

// 🍴 Меню
GET /items/menu_items?filter[status][_eq]=published
GET /items/menu_items/:id

// 🪑 Столики
GET /items/tables?filter[status][_eq]=active

// 📅 Бронирования
GET  /items/reservations?filter[user_created][_eq]=:userId
POST /items/reservations   # Создание брони (требуется авторизация)
PATCH /items/reservations/:id
DELETE /items/reservations/:id
```

### Пример запроса из booking.js

```javascript
const token = getAuthToken();
const response = await fetch(`${getApiUrl()}/items/reservations`, {
    method: 'POST',
    headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${token}` 
    },
    body: JSON.stringify({
        date: '2026-04-15',
        time: '19:00:00',
        table: 'table_123',
        party_size: 4,
        total: 2500,
        status: 'pending'
    })
});
```

## 🛠 Разработка

### 🧪 Тестирование локально

```bash
# Проверка синтаксиса JS (опционально)
npx eslint .

# Запуск с кастомным API
# Создайте config.js в корне:
echo "window.API_CONFIG = { DIRECTUS_URL: 'http://localhost:8055' };" > config.js
```

### 🎨 Стили

Все цвета вынесены в CSS-переменные в `style.css`
Для изменения темы отредактируйте `:root { --color-primary: #D4A373; ... }`

### 🌍 Мультиязычность

Проект пока на русском. Для добавления других языков:
1. Вынесите все тексты в отдельный `i18n.js`
2. Добавьте переключатель языка в навбар
3. Загружайте нужный словарь при инициализации
