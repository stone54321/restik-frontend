# Инструкция по настройке Directus для меню Restik

## 1. Установка Directus

Если у вас еще не установлен Directus, выполните:

```bash
# Глобальная установка
npm install -g directus

# Создание нового проекта
npx create-directus-project my-directus

# Запуск
cd my-directus
npx directus start
```

## 2. Настройка коллекции menu_items

### Создание коллекции

1. Зайдите в админ-панель Directus (обычно http://localhost:8055)
2. Перейдите в Settings → Data Model
3. Нажмите "+" для создания новой коллекции
4. Назовите коллекцию `menu_items`

### Поля коллекции

Добавьте следующие поля:

| Поле | Тип | Обязательно | Описание |
|------|-----|-------------|----------|
| name | Input | Да | Название блюда |
| description | Textarea | Да | Описание блюда |
| price | Integer | Да | Цена в рублях |
| category | Select | Да | Категория блюда |
| image | File | Нет | Изображение блюда |
| status | Select | Да | Статус (published/draft) |

### Настройка поля category

Для поля `category` выберите тип "Select" и добавьте опции:
- `appetizers` - Закуски
- `soups` - Супы  
- `mains` - Основные блюда
- `desserts` - Десерты
- `drinks` - Напитки

### Настройка поля status

Для поля `status` выберите тип "Select" и добавьте опции:
- `published` - Опубликовано
- `draft` - Черновик

Установите значение по умолчанию `draft`.

## 3. Настройка прав доступа

1. Перейдите в Settings → Roles & Permissions
2. Откройте роль "Public"
3. Найдите коллекцию `menu_items`
4. Установите права:
   - Read: ✅ (включено)
   - Create: ❌
   - Update: ❌
   - Delete: ❌

## 5. Настройка CORS (если необходимо)

Если фронтенд находится на другом домене/порту:

1. Перейдите в Settings → Project Settings → CORS
2. Добавьте URL вашего фронтенда (например: `http://localhost:3000`)
3. Установите галочки:
   - ✅ GET
   - ✅ POST
   - ✅ PUT
   - ✅ DELETE
   - ✅ PATCH

## 6. Проверка API

Проверьте работу API через браузер:
```
http://localhost:8055/items/menu_items?status=published
```

Должен вернуть JSON с вашими блюдами.

## 7. Настройка фронтенда

В файле `menu.js` убедитесь, что указан правильный URL:

```javascript
const DIRECTUS_URL = 'http://localhost:8055'; // Ваш URL Directus
```

## 8. Дополнительные настройки

### Кэширование изображений

Для оптимизации загрузки изображений:

1. В настройках Directus включите кэширование
2. Настройте CDN для изображений, если необходимо

### SEO оптимизация

Добавьте поля для SEO:
- `seo_title` - Title для страницы блюда
- `seo_description` - Meta описание
- `seo_keywords` - Ключевые слова

### Мультивалютность

Для поддержки нескольких валют:
- Добавьте поле `currency` (select: RUB, USD, EUR)
- Добавьте поле `price_usd` и `price_eur`
- Модифицируйте JS для переключения валют

## 9. Тестирование

1. Запустите Directus: `npx directus start`
2. Откройте `frontend/menu/menu.html` в браузере
3. Проверьте загрузку блюд из Directus
4. Протестируйте фильтры по категориям

## 10. Возможные проблемы

### Ошибка CORS
- Проверьте настройки CORS в Directus
- Убедитесь, что фронтенд URL добавлен в белый список

### Нет данных
- Проверьте, что у блюд статус `published`
- Убедитесь, что у Public роли есть права на чтение

### Изображения не загружаются
- Проверьте права доступа к файлам
- Убедитесь, что изображения загружены в Directus

## 11. Продвинутые функции

### Поиск блюд
Добавьте поле `search_keywords` для улучшения поиска.

### Акции и скидки
Добавьте поля:
- `discount_price` - Цена со скидкой
- `discount_percentage` - Процент скидки
- `discount_valid_until` - Дата окончания акции

### Состав и аллергены
Добавьте поля:
- `ingredients` - Состав (JSON)
- `allergens` - Аллергены (множественный select)
- `calories` - Калорийность
- `cooking_time` - Время приготовления

Эта инструкция поможет полностью интегрировать меню с Directus и обеспечить динамическое управление блюдами через админ-панель.
